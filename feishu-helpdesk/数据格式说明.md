# 📊 数据格式说明

## 多维表格数据结构

从飞书多维表格读取的工单数据格式：

```json
{
  "name": "工单标题/问题描述",
  "ticket_id": 7400576031599525892,
  "msg": [
    {
      "user_name": "用户名称",
      "date": "2024-08-08 09:55:37",
      "content": "消息内容"
    }
  ]
}
```

---

## 字段说明

### 工单对象

| 字段 | 类型 | 说明 | 来源 |
|------|------|------|------|
| `name` | string | 工单标题/问题描述 | 多维表格 |
| `ticket_id` | number | 工单 ID | 多维表格 |
| `msg` | array | 聊天记录数组 | 多维表格 |

### 消息对象

| 字段 | 类型 | 说明 |
|------|------|------|
| `user_name` | string | 发送者姓名 |
| `date` | string | 发送时间（格式：YYYY-MM-DD HH:mm:ss） |
| `content` | string | 消息内容（可能包含图片、@等） |

---

## 数据解析流程

```
原始 JSON 数据
    ↓
ticket-parser.js 解析
    ↓
提取关键信息
    ↓
生成总结文档
```

### 解析逻辑

1. **工单号** ← `ticket_id`
2. **日期** ← `msg[0].date` 的日期部分
3. **提出人** ← `msg[0].user_name`（第一条消息发送者）
4. **对接人** ← 第一个回复消息的人（非提出人）
5. **问题描述** ← `name` 字段（优先）或早期消息
6. **问题根因** ← 后期消息中的结论性内容
7. **解决方案** ← 后期消息中的处理建议
8. **问题类别** ← 智能分类器分析
9. **问题标签** ← 智能分类器分析

## 特殊处理

### 图片消息
```json
{ "content": "<img src=\"/open-apis/...\"/>" }
```
→ 自动过滤，不作为问题描述或根因

### @消息
```json
{ "content": "<at user_id=\"xxx\">@用户</at> 消息内容" }
```
→ 保留文本内容，用于分析

### 空消息
```json
{ "content": "" }
```
→ 跳过

## 配置多维表格

### 1. 创建多维表格

在飞书中创建多维表格，至少包含以下字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 工单标题 | 文本 | 对应 `name` |
| 工单 ID | 数字 | 对应 `ticket_id` |
| 聊天记录 | 富文本/JSON | 对应 `msg` 数组 |
| 工单日期 | 日期 | 自动填充 |
| 状态 | 单选 | 已结束/处理中 |

### 2. 获取配置信息

编辑 `config/bitable-config.json`：

```json
{
  "bitableAppToken": "bascnXXX123",
  "bitableTableId": "tblXXX456",
  "fieldMapping": {
    "name": "工单标题",
    "ticket_id": "工单 ID",
    "msg": "聊天记录",
    "status": "状态"
  }
}
```
