# 🤖 智能分类说明

## 功能概述

系统会根据工单内容**自动分析**问题类别和问题标签，无需手动填写。

---

## 📋 问题类别（9 种）

系统从以下类别中自动选择最匹配的一个：

| 类别 | 关键词示例 |
|------|-----------|
| **MA 画布** | MA、画布、营销画布、客户旅程、触点、营销自动化 |
| **智能外呼** | 外呼、呼叫、电话、拨打、重呼、呼叫任务、接通率 |
| **bot（话术）** | bot、话术、机器人、自动回复、问答、对话、客服 bot |
| **智能体 bot（话术）** | 智能体、AI bot、AI 话术、语义理解、意图识别、NLP |
| **对账问题** | 对账、账务、结算、金额、账目、核对、差异、账单 |
| **数据异常** | 数据、异常、不一致、错误、缺失、重复、脏数据 |
| **私有化问题** | 私有化、本地部署、私有云、本地化、on-premise |
| **海外问题** | 海外、国际、境外、跨境、多语言、时区、国际化 |
| **政务问题** | 政务、政府、政务云、政务系统、政府部门、信创 |

---

## 🏷️ 问题标签（5 种）

系统根据问题根因自动匹配标签：

| 标签 | 关键词示例 |
|------|-----------|
| **系统处理逻辑缺陷** | 逻辑、判断、条件、流程、规则、算法、计算错误、状态机 |
| **接口与集成故障** | 接口、API、集成、第三方、对接、调用失败、超时、网络 |
| **数据质量与校验缺失** | 校验、验证、数据质量、格式、必填、数据类型、数据清洗 |
| **系统可靠性与可观测性不足** | 监控、日志、告警、性能、稳定性、可用性、故障、宕机 |
| **操作不当** | 操作、配置、设置、使用、误操作、人为、配置错误、不规范 |

## 🔍 分类示例

### 示例 1
```
问题描述：用户报告智能外呼任务结束后总有少量数据无法完成
问题根因：客户在下午 4-5 点非工作时间偷偷往任务中添加新数据

→ 问题类别：智能外呼
→ 问题标签：操作不当
```

### 示例 2
```
问题描述：MA 画布流程执行到某个节点时卡住，无法继续
问题根因：节点之间的数据传递逻辑存在缺陷

→ 问题类别：MA 画布
→ 问题标签：系统处理逻辑缺陷
```

### 示例 3
```
问题描述：bot 机器人回复内容不正确
问题根因：话术配置错误，操作人员配置了错误的回复规则

→ 问题类别：bot（话术）
→ 问题标签：操作不当
```

### 示例 4
```
问题描述：海外客户反馈系统访问慢
问题根因：海外节点网络延迟高，缺少 CDN 加速

→ 问题类别：海外问题
→ 问题标签：接口与集成故障
```

## ⚙️ 工作原理

### 1. 关键词匹配
系统内置关键词库，匹配问题描述和根因中的关键词。

### 2. 评分排序
每个类别/标签根据匹配到的关键词数量评分，得分最高的获胜。

### 3. 智能 fallback
- 如果多维表格中已填写类别/标签，优先使用手动填写的
- 如果未填写，自动分析生成

---

## 📊 分类逻辑

```javascript
问题描述 + 问题根因
    ↓
关键词匹配（不区分大小写）
    ↓
计算每个类别的匹配分数
    ↓
选择得分最高的类别
    ↓
输出：问题类别 + 问题标签
```

## 🔧 自定义分类规则

编辑 `scripts/classifier.js` 可以自定义分类规则：

```javascript
const CATEGORY_KEYWORDS = {
  '你的类别': ['关键词 1', '关键词 2', ...],
  // ...
};

const TAG_KEYWORDS = {
  '你的标签': ['关键词 1', '关键词 2', ...],
  // ...
};
```

---

## ⚠️ 注意事项

1. **类别互斥**：每个工单只会有一个问题类别
2. **标签可多选**：最多匹配 2 个标签
3. **未分类处理**：如果无法匹配，返回"未分类"
4. **手动优先**：多维表格中已填写的类别/标签优先使用
