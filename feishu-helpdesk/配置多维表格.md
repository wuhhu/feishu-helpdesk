# 📊 配置飞书多维表格

## 数据来源说明

工单总结数据来自**飞书多维表格（Bitable）**，不是服务台 API。

---

## 🔑 获取配置信息

### 1. 打开多维表格

在飞书中打开你的工单管理多维表格。

### 2. 获取 AppToken

从 URL 中提取：
```
https://your-company.feishu.cn/base/bascnXXX123?table=tblXXX456
                                      ↑ 这就是 AppToken
```

**AppToken:** `bascnXXX123`

### 3. 获取 TableId

从 URL 中提取：
```
https://your-company.feishu.cn/base/bascnXXX123?table=tblXXX456
                                                      ↑ 这就是 TableId
```

**TableId:** `tblXXX456`

---

## ⚙️ 配置步骤

### 1. 编辑配置文件

```bash
vi /home/admin/.openclaw/workspace/feishu-helpdesk/config/bitable-config.json
```

### 2. 填入配置

```json
{
  "bitableAppToken": "bascnXXX123",
  "bitableTableId": "tblXXX456",
  "dateField": "工单日期",
  "statusField": "状态",
  "closedStatus": "已结束",
  "fieldMapping": {
    "ticketId": "工单号",
    "date": "工单日期",
    "requester": "工单提出人",
    "assignee": "工单对接人",
    "problemDesc": "问题描述",
    "rootCause": "问题根因",
    "solution": "解决方案",
    "category": "问题类别",
    "tags": "问题标签",
    "status": "状态"
  }
}
```

---

## 📋 多维表格字段要求

你的多维表格应包含以下字段：

| 字段名 | 类型 | 说明 | 必填 |
|--------|------|------|------|
| 工单号 | 文本 | 唯一标识 | ✅ |
| 工单日期 | 日期 | 提出日期 | ✅ |
| 工单提出人 | 文本/人员 | 申请人 | ✅ |
| 工单对接人 | 文本/人员 | 处理人 | ✅ |
| 问题描述 | 多行文本 | 问题详情 | ✅ |
| 问题根因 | 多行文本 | 原因分析 | ✅ |
| 解决方案 | 多行文本 | 处理方案 | ✅ |
| 问题类别 | 单选 | 分类 | ✅ |
| 问题标签 | 多选 | 标签 | ⭕ |
| 状态 | 单选 | 已结束/处理中 | ✅ |

## ⚠️ 权限说明

### 当前权限
- ✅ `aily:file:read` - 读取飞书文件
- ✅ `aily:file:write` - 写入飞书文件

### 阶段二需要
- ⏳ `bitable:app:read` - 读取多维表格数据
- ⏳ `bitable:app:write` - 写入多维表格
